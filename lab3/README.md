# Лабораторная работа №3: Модели данных и административный интерфейс

## Описание проекта
В рамках данной работы реализована система управления контентом для блога. Основной упор сделан на работу с базами данных через ORM Django и настройку профессиональной панели администратора.

## Основные возможности
* **Моделирование данных**: Создана модель `Article`, включающая поля для заголовка, автора (связь с системной моделью `User`), текста и даты публикации.
* **Кастомизация админки**: Настроен класс `ArticleAdmin` для отображения списка статей с предпросмотром текста (метод `get_excerpt`) и метаданными.
* **Динамические шаблоны**: Реализована страница архива, которая автоматически подтягивает все записи из базы данных и отображает их в цикле.

## Структура папок
* **blog/** — настройки проекта (`settings.py`, `urls.py`).
* **articles/** — приложение блога.
  * **models.py** — описание структуры таблицы статей.
  * **admin.py** — конфигурация отображения статей в панели управления.
  * **views.py** — логика извлечения записей из БД.
  * **templates/archive.html** — динамическая верстка списка постов.

## Технические особенности (Django 5.x)
Код адаптирован под современные стандарты разработки:
* Вместо устаревшего `__unicode__` используется метод `__str__`.
* Поле автора `ForeignKey` настроено с обязательным параметром `on_delete=models.CASCADE`.
* Для подключения статики (логотипа) используется тег `{% load static %}`.

## Инструкция по запуску

1. Перейдите в папку проекта:
```bash
cd blog
```
2. Выполните миграции для создания таблиц в БД:
```bash
python manage.py makemigrations
python manage.py migrate
```
3. Создайте учетную запись администратора:
```bash
python manage.py createsuperuser
```
4. Запустите сервер:
```bash 
python manage.py runserver
```
После запуска архив статей доступен на главной странице http://127.0.0.1:8000/, а управление статьями — в админке http://127.0.0.1:8000/admin/.
