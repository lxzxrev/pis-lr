# Лабораторная работа №4: Создание страницы отдельной записи

## Описание проекта
В данной работе реализована функциональность просмотра полной версии статьи. Основное внимание уделено динамической маршрутизации с использованием регулярных выражений и продвинутой верстке по специфическому макету.

## Основные возможности
* **Динамические URL**: Использование `re_path` и именованных групп `(?P<article_id>\d+)` для извлечения ID статьи прямо из адресной строки.
* **Обработка ошибок**: Реализована ручная проверка существования записи с генерацией системной ошибки `Http404` (Page Not Found).
* **Сложная верстка (CSS)**:
    * **Архив (Главная)**: Логотип выровнен по левому краю.
    * **Статья**: Логотип центрирован по горизонтали.
    * **Навигация**: Кнопка «Все статьи» на странице записи прижата к правому краю и выровнена по высоте относительно логотипа.
    * **Типографика**: Увеличенный размер шрифта (22px/36px), курсив для дат и подчеркивание для заголовков.

## Структура проекта
* **blog/urls.py** — конфигурация маршрутов с использованием регулярных выражений.
* **articles/views.py** — функция `get_article` с логикой `try-except`.
* **articles/templates/**:
    * `archive.html` — обновленный список статей со ссылками.
    * `article.html` — новый шаблон для отображения одной записи.
* **articles/static/css/article.css** — стили, обеспечивающие разное позиционирование элементов для разных страниц.

## Инструкция по запуску

1. Перейдите в папку проекта:
   ```bash
   cd blog
   ```
2. Примените миграции (если база данных еще не настроена):
   ```bash
   python manage.py migrate
   ```
3. Запустите сервер:
   ```bash
   python manage.py runserver
   ```
4. Доступ к страницам:
- Главная: http://127.0.0.1:8000/
- Страница статьи: Кликните по подчеркнутому заголовку любой статьи в списке.
