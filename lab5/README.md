# Лабораторная работа №5: Создание форм и обработка POST-запросов

## Описание проекта
В данной работе реализован функционал создания новых статей пользователями непосредственно через интерфейс сайта. Проект перешел от режима «только чтение» к полноценному взаимодействию с базой данных через веб-формы.

## Основные возможности
* **Обработка форм**: Реализовано представление `create_post`, которое разграничивает GET-запросы (отображение формы) и POST-запросы (сохранение данных).
* **Безопасность (Security)**:
    * **Аутентификация**: Добавлен запрет на создание постов для анонимных пользователей (`is_authenticated`).
    * **CSRF-защита**: Использование тега `{% csrf_token %}` для предотвращения атак типа Cross-Site Request Forgery.
* **Валидация и логика**:
    * Проверка на заполненность обязательных полей (Заголовок и Текст).
    * **Уникальность заголовка**: Реализована проверка через базу данных, исключающая создание статей с одинаковыми названиями.
    * Перенаправление (Redirect) на страницу созданной статьи после успешного сохранения.

## Структура проекта
* **articles/views.py** — логика создания поста и валидация данных.
* **articles/templates/create_post.html** — специализированный шаблон с тегами `<form>`, `<input>` и `<textarea>`.
* **articles/static/css/article.css** — стили для адаптивной и удобной формы ввода.

## Технические детали (2026 Edition)
* **Django**: 6.0.3.
* **Python**: 3.14.
* Использование `redirect` и именованных URL-адресов для гибкой навигации.

## Инструкция по запуску

1. Перейдите в директорию `blog`:
```bash
cd blog
```
2. Убедитесь, что вы авторизованы в системе (создайте суперпользователя, если это новый проект):
```bash
python manage.py createsuperuser
```
3. Запустите сервер:
```bash
python manage.py runserver
```
4. Форма создания статьи доступна по адресу:
http://127.0.0.1:8000/article/new/
