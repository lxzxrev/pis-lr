# Лабораторная работа №6: Регистрация и аутентификация пользователей

## Описание проекта
Шестой этап разработки превратил проект из простого блога в полноценную многопользовательскую платформу. В данной работе реализована система управления учетными записями, позволяющая пользователям регистрироваться, входить в систему и выходить из неё.

## Основные возможности
* **Регистрация (Sign Up)**: Создание новых аккаунтов с использованием метода `User.objects.create_user`, обеспечивающего безопасное хеширование паролей.
* **Аутентификация (Sign In)**: Проверка подлинности данных через функцию `authenticate` и создание сессии через `login`.
* **Динамическая навигация (Smart Header)**:
    * Для анонимных пользователей отображаются кнопки **Вход** и **Регистрация**.
    * Для авторизованных — приветствие (имя пользователя), кнопка **Написать статью** и **Выход**.
* **Исправленная верстка (Flexbox & Absolute Positioning)**:
    * Решена проблема наложения кнопок в шапке.
    * Логотип центрирован на страницах форм и чтения статей.
    * Ссылка «Все статьи» выровнена по правому краю и центрирована по высоте относительно логотипа.
    * Все поля ввода (логин, email, пароль) имеют строго одинаковую ширину благодаря `box-sizing: border-box`.

## Структура проекта
* **articles/views.py** — логика `register`, `login_user` и `logout_user`.
* **articles/templates/**:
    * `registration.html` — форма создания нового аккаунта.
    * `login.html` — форма входа.
    * `archive.html` — обновленная шапка с условиями `{% if user.is_authenticated %}`.
* **articles/static/css/article.css** — финальные стили для позиционирования элементов шапки и унификации полей ввода.

## Технические детали (2026 Edition)
* **Django**: 6.0.3.
* **Python**: 3.14.
* Использование `position: relative` и `position: absolute` для точного позиционирования кнопки возврата на главную.

## Инструкция по запуску

1. Перейдите в папку проекта:
   ```bash
   cd blog
   ```
2. Примените миграции (Django создаст системные таблицы для пользователей):
   ```bash
   python manage.py migrate
   ```
3. Запустите сервер:
   ```bash
   python manage.py runserver
   ```
4. Тестирование:
- Зарегистрируйте нового пользователя.
- Убедитесь, что после входа в шапке появилось ваше имя и кнопка «Выход».
- Проверьте, что кнопка «Написать статью» доступна только после авторизации.
